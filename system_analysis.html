Step 2

site critique using Nielsen's heuristics
1.Visibility of system status
the visibility is pretty good, it does give 'breadcrumbs' which is good for backtracking
2.Match between system and the real world
the menus are clearly labeled but it does not appear to have alternate language options
3.User control and freedom
with the breadcrumbs it is easy to go back to the previous page, the bar with links to home is available on every page
4.Consistency and standards
The system is very consistent, all pages have the same layout.
5.Error prevention
there are not many places to click which limits the chances of selecting the wrong thing.
6.Recognition rather than recall
the recognition is good, things are clearly labeled
7.Flexibility and efficiency of use
there are no noticable shortcuts or customization for frequent actions
8.Aesthetic and minimalist design
definitely is a minimalistic design making it easy to read and use
9.Help users recognize, diagnose and recover from errors
when an image doesn't load it shows a blue square with a question mark but there is nothing to do to fix that error when there are no results for your search are clear
10.Help and documentation
there are no options for help, FAQ, etc. but there is a contact page which you could use to reach out to the departments.

Step 3

Strengths

  • Simple, minimalist design
  • Easy to navigate

Weaknesses

  • VERY slow
  • Search is not descriptive

The simple minimalist design meets the standards of 8 which involves aesthetic. The system is easy to navigate which meets the standards of 4 consistency and 7 efficiency.
The system is very slow which goes against 7 efficiency. The search results are not descriptive which goes against error prevention because you have to do a lot of trial and error to find the result you are looking for.

Part 2

Step 5

comparison of part 1 and 2 results. What similarities, differences were there. Also include some suggstions. The system being very slow was the main thing noticed in part 1 and 2. A small experiment proved that it did not matter what browser you used, or computer you were on, it was equally as slow no matter what. Another similarity was the lack of information in the search results. It is difficult to select which result would best match your search when there is no context given. This would be particularly difficult if you were searching for anything that gives a lot of results. The option to narrow down the results helps, but when you filter the results it removes the 'this collection only' filter which is very annoying. It would be very helpful if there were more options to narrow your search, and if the results could show you what context the word is used in. When searching for information about World War II and Vietnam, it was much easier to some results because there are distinct date ranges that would be relevant for this search. You know that you aren't going to be looking in the issues from 1925 for articles about World War II because it hasn't happened yet. As the Catholic University Archives switches over to JSTOR Forum as host for our digital collections, we hope that the user experience will be easier and the program will be faster since that is a piece of feedback we receive often from researchers. Hopefully there will be more accessibility features available while keeping the simple, clean design that is enjoyed in Islandora.
